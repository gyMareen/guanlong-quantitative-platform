# 官龙量化一体化交易平台 - 系统架构

## 1. 系统概述

官龙量化一体化交易平台是一个集成订单采集、量化策略、交易中台的本地化交易系统。系统通过 Appium 从富途 App 采集调仓信号，结合 Python 量化策略生成交易信号，最终由 Java 交易中台执行风控和下单。

## 5. 数据流设计

### 5.1 采集数据流

```
富途 App → Appium 定位 → 解析行数据
    → 去重（日期/批次）
    → PostgreSQL (rebalance_raw)
    → Kafka (futu.rebalance.raw)
```

### 5.2 量化数据流

```
历史数据 → 因子计算 → 策略信号
    → 组合融合 → 权重归一
    → PostgreSQL (rebalance_qs)
    → Kafka (futu.rebalance.qs)
```

### 5.3 交易数据流

```
Kafka (raw/qs) → 信号消费 → 信号归并
    → 差值计算 → 风控校验
    → LongPort 下单 → 订单入库
    → Kafka (futu.trade.events)
```

## 6. 风控设计

### 6.1 风控规则

| 规则 | 参数 | 说明 |
|------|------|------|
| 单票仓位上限 | 20% | 单只股票不超过总资产 20% |
| 组合仓位上限 | 95% | 总仓位不超过 95% |
| 单日亏损上限 | 5% | 单日亏损超过 5% 停止交易 |
| 价格偏离保护 | 4% | 下单价格偏离超过 4% 拒绝 |
| 最小交易额 | $20 | 差额小于 20 美元不执行 |
| 清仓阈值 | 0 | 目标仓位为 0 时执行清仓 |

### 6.2 风控流程

```
信号 → 仓位检查 → 亏损检查 → 价格检查 → 金额检查 → 执行/拒绝
```

## 8. 监控设计

### 8.1 指标采集

- 应用指标：请求数、响应时间、错误率
- Kafka 指标：消息积压、吞吐量
- 业务指标：订单数、成交额、盈亏

### 8.2 告警规则

- Kafka 消费延迟 > 1000
- 订单失败率 > 10%
- 单日亏损 > 3%
- 系统错误率 > 5%
